Projects

PlainSight (2020)




PlainSight is my first published browser extension. I first had the idea for this project a few years ago but at the time I had no idea how to go about creating a browser extension and had some other projects going on anyway. When I remembered the concept, I decided that I was going to figure out how to create a Chrome extension. As it turns out, it's almost the exact same as creating a Firefox extension or Edge extension, so I ended up supporting all three!

The basic idea behind this project is that it allows people to communicate secretly over public channels (Twitch chat, Facebook group messages/statuses, Twitter, etc.). The user enters a password into the extension popup which is then converted into a 192-bit key via the scrypt password derivation algorithm. This key is then used to symmetrically encrypt a message using AES in CBC mode with a randomized initialization vector (which is then combined with the encrypted message). This ensures that the encrypted output is always different even if the input message is repeated. This encrypted message can be posted anywhere on the web and any other user with the extension installed that is using the same password will see the decrypted message. There are a few hotkeys and context menu options that were added to improve the fluidity and overall user experience of the extension.

The real meat of this project was created with vanilla JavaScript which is supported with some basic HTML and CSS (primarily for the extension popup). I've always been interested in cryptography so this project was especially fun and has inspired some other projects that will probably be listed here soon.

Alteration (2018)




Alteration is a survival-horror game created as a final project for a Computer Graphics course that I took during my last year at Georgia Southern. This was a group project with Jared Dean and Najee Simmons. We created this game in Unity and used C# for all of the scripting.

This game was a lot of fun to create. It contains five main chambers and a maze, each with their own puzzles and challenges to overcome. The very first room that the player spawns in (essentially just four walls with torches and a door) was built using a Unity tutorial, but the rest of the game was created by attempting to replicate features from games that the other group members and I had played in the past. This was the most fun aspect of this project as we had to come up with logic to create monster mechanics, weapons, checkpoints, the pause menu, and more.

A demo of this game is available here and you can view the source code here. This project was a little on the large side and we weren't able to hammer out all of the bugs before submission, but it's still functional from beginning to end. It was compiled to run on Windows and will not execute on other operating systems. You can run it by extracting the ZIP file and executing gametest.exe. If you decide to try it out, good luck, you'll need it.

Gaming Control Using Gaze and Gestures (2018)




Gaming Control Using Gaze and Gestures was the capstone project for the Software Engineering course I took during my last year at Georgia Southern. This was a group project with Danny Burer, Rosalie Fragnito, John Lillard, and Kaden Graham. We created this project in Unity and used C# for all of the scripting.

The purpose of this project was to expose the members of the group to the SDLC while working with a client, Dr. Pradipta De (another professor at the University). Dr. De was curious about the possibility of using eye tracking hardware to monitor the attentiveness of students during lectures and one of our goals during the course of this project was to evaluate the feasibility of this idea. We completed several deliverables throughout the life of this project including a feasibility report, a software requirements and specification document, a software design document, a software testing and quality assurance document, and a final notebook tying it all together.

For this project, we were supplied with a Tobii 4C Eye Tracker which has a Unity SDK that we used to implement eye controls into an open source game that we found, a recreation of the popular app Fruit Ninja. The game was pretty basic and we ended up adding several features to it on top of new controls (bombs, different fruit, QoL improvements, an advanced replay system connected to an AWS database instance, analytics, main menu / pause menu, etc.).

Unfortunately due to the nature of the project, the source code is not available (stored in a private repository). The graphic we used at the Georgia Southern CEIT (now CEC) Annual Student Research Symposium, however, is available here.

Humans vs. Zombies at Georgia Southern Website (2014 - 2016)




The website I created for Humans vs. Zombies at Georgia Southern was the first real web development project that I worked on. I was approached by the president of the organization and was asked if I could build a website that better fit their needs. I saw this as an opportunity to teach myself a new language and began creating the website that is still in use by the organization today. This project was created using PHP, JavaScript, HTML/CSS, and SQL. Since I saw this as a learning opportunity for PHP, I neglected to use any development frameworks when making this website.

The website is designed to support the game, which is essentially a big game of tag. Each player has a "state" of human, zombie, or original zombie and is assigned a secret ID that they carry with them. The ID of a human is given to a zombie if that human is tagged, and the human then becomes a zombie. The content of the website is dependant on the state of the user. Everyone can see who is a human and who is a zombie, but zombies can enter player IDs into the site to "kill" a human, and humans can send out text blasts requesting help if they are surrounded by zombies. There is also state-specific information distributed by the moderators of the game that only players of the correct state can see on the website.

There is also an administration panel that only game moderators can see. This panel allows moderators to send email blasts, update the state of the game, post news articles, change the state of players, send email blasts, control "missions" using text blasts, and much more.
